# Agents Rules

1. Использовать BEM-lite: именовать блоки и элементы по BEM (`block`, `block__element`), а модификаторы добавлять только при необходимости (`block--modifier`).
2. Не усложнять селекторы: избегать глубокой вложенности и длинных цепочек.
3. Для компонентных стилей предпочитать локальную изоляцию (`scoped`), а значения выносить в CSS-переменные.
4. Новые UI-части выносить в отдельные компоненты, а страницу собирать из них.
5. Не писать грязный CSS: избегать магических чисел и дублирования, общие размеры и отступы выносить в осмысленные CSS-переменные.
6. Объёмные структуры, общие контролы и нетривиальную логику выносить из страниц и `app.vue` в отдельные компоненты и composables.
7. Каждый блок страницы выносить в отдельный компонент, а страницу использовать как композиционный слой.
